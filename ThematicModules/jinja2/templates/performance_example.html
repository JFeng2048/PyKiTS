1. 基础性能优化技术

1.1 上下文数据优化

在模板渲染前预处理数据，可以显著提高渲染性能。下面是一个优化前和优化后的示例对比：

优化原则：尽量在视图函数中完成数据处理，而不是在模板中进行复杂计算。

优化前：在模板中处理数据

<!-- 不推荐的方式 -->
{% for user in users %}
  <div class="user-item"
    {% if user.last_login > (now - timedelta(days=30)) %}
      style="background-color: #e8f4f8;"
    {% endif %}
  >
    <h4>{{ user.name|title }}</h4>
    <p>角色: {{ user.roles|join(', ', attribute='name') }}</p>
    <p>登录次数: {{ user.logins|length }}</p>
    <p>上次登录: {{ user.last_login.strftime('%Y-%m-%d %H:%M') }}</p>
  </div>
{% endfor %}

优化后：预处理器数据

<!-- 推荐的方式 -->
{% for user in processed_users %}
  <div class="user-item {{ 'recently-active' if user.is_recently_active else '' }}">
    <h4>{{ user.display_name }}</h4>
    <p>角色: {{ user.role_names }}</p>
    <p>登录次数: {{ user.login_count }}</p>
    <p>上次登录: {{ user.last_login_display }}</p>
  </div>
{% endfor %}
性能提升：通过预处理数据，可以减少模板中的计算量，提高渲染速度约 30%-50%。

示例数据展示

{% for user in processed_users[:5] %}

{{ user.display_name }}

角色: {{ user.role_names }}

登录次数: {{ user.login_count }}

上次登录: {{ user.last_login_display }}

{% endfor %}

2. 缓存优化技术

2.1 页面片段缓存

缓存页面中不经常变化的部分，可以显著减少渲染时间。下面是一个使用缓存块的示例：

<!-- 缓存侧边栏30分钟 -->
{% cache 'sidebar', timeout=1800 %}
  <div class="sidebar">
    <h3>热门文章</h3>
    <ul>
      {% for article in popular_articles %}
        <li><a href="{{ article.url }}">{{ article.title }}</a></li>
      {% endfor %}
    </ul>
    
    <h3>分类导航</h3>
    <ul>
      {% for category in categories %}
        <li><a href="{{ category.url }}">{{ category.name }}</a> ({{ category.count }})</li>
      {% endfor %}
    </ul>
  </div>
{% endcache %}
注意事项：缓存块需要通过扩展实现，例如 Flask-Caching 或自定义的缓存扩展。缓存键应该包含所有影响缓存内容的变量。

热门文章（模拟缓存内容）

    {% for article in popular_articles %}
  • {{ article.title }} ({{ article.views }} 次浏览)
    • {% endfor %}

缓存生成时间: {{ cache_time }}

3. 模板结构优化

3.1 简化模板逻辑

复杂的模板逻辑会显著降低渲染性能。下面是一些简化模板逻辑的技巧：

避免复杂的嵌套条件

<!-- 优化前 -->
{% if user.is_admin %}
  <div class="admin-panel">
    {% if user.has_permission('view_all') %}
      <button>查看全部</button>
    {% endif %}
    {% if user.has_permission('edit_all') %}
      <button>编辑全部</button>
    {% endif %}
  </div>
{% elif user.is_manager %}
  <div class="manager-panel">
    {% if user.team_id == current_team.id %}
      <button>管理团队</button>
    {% endif %}
  </div>
{% endif %}

<!-- 优化后 -->
{% if user.is_admin %}
  <div class="admin-panel">
    {{ admin_buttons }}
  </div>
{% elif user.is_manager %}
  <div class="manager-panel">
    {{ manager_buttons }}
  </div>
{% endif %}

3.2 合理使用宏

宏可以提高代码复用性，但过度使用也会影响性能。下面是一个平衡的示例：

<!-- 定义宏 -->
{% macro render_pagination(pagination) %}
  <div class="pagination">
    {% if pagination.has_prev %}
      <a href="?page={{ pagination.prev_num }}">上一页</a>
    {% endif %}
    
    {% for page in pagination.iter_pages() %}
      {% if page %}
        {% if page == pagination.page %}
          <strong>{{ page }}</strong>
        {% else %}
          <a href="?page={{ page }}">{{ page }}</a>
        {% endif %}
      {% else %}
        <span class="ellipsis">...</span>
      {% endif %}
    {% endfor %}
    
    {% if pagination.has_next %}
      <a href="?page={{ pagination.next_num }}">下一页</a>
    {% endif %}
  </div>
{% endmacro %}

<!-- 使用宏 -->
{{ render_pagination(pagination) }}
上一页 1 2 3 ... 10 下一页

4. 循环优化技术

4.1 减少循环内的计算

循环内的重复计算是性能杀手。下面是优化前后的对比：

<!-- 优化前 -->
{% for item in items %}
  <div class="item">
    <h4>{{ item.name|title }}</h4>
    <p>总价: {{ item.price * item.quantity|round(2) }}</p>
    <p>折扣: {{ item.price * item.discount|percent }}</p>
    <p>库存状态: {{ '有货' if item.stock > 0 else '缺货' }}</p>
    {% if item.category in featured_categories %}
      <span class="featured">热门商品</span>
    {% endif %}
  </div>
{% endfor %}

<!-- 优化后 -->
{% for item in processed_items %}
  <div class="item {{ 'featured' if item.is_featured else '' }}">
    <h4>{{ item.display_name }}</h4>
    <p>总价: {{ item.total_price }}</p>
    <p>折扣: {{ item.discount_percent }}</p>
    <p>库存状态: {{ item.stock_status }}</p>
    {{ item.featured_badge }}
  </div>
{% endfor %}
性能提升：对于大型列表，将循环内的计算移至视图函数可以提高渲染速度 50%-80%。
{% for item in processed_items[:4] %}

{{ item.display_name }}

总价: {{ item.total_price }}

折扣: {{ item.discount_percent }}

库存状态: {{ item.stock_status }}

{% if item.is_featured %} 热门商品 {% endif %}
{% endfor %}

5. 高级性能优化技术

5.1 延迟加载与分页

对于大型数据集，使用延迟加载和分页可以显著提高性能：

<!-- 分页显示大型数据集 -->
<div class="large-dataset">
  {% for item in paginated_items %}
    <div class="dataset-item">
      {{ item.content }}
    </div>
  {% endfor %}
  
  {{ render_pagination(pagination) }}
</div>

<!-- 延迟加载更多内容 -->
<div id="lazy-load-container">
  {% for item in initial_items %}
    <div class="lazy-item">
      {{ item.content }}
    </div>
  {% endfor %}
</div>
<button id="load-more" data-next-page="{{ next_page }}">加载更多</button>

<script>
  document.getElementById('load-more').addEventListener('click', function() {
    const nextPage = this.getAttribute('data-next-page');
    if (nextPage) {
      fetch('/load-more?page=' + nextPage)
        .then(response => response.text())
        .then(html => {
          const container = document.getElementById('lazy-load-container');
          container.insertAdjacentHTML('beforeend', html);
          // 更新下一页链接
          this.setAttribute('data-next-page', ...);
        });
    }
  });
</script>

5.2 选择性渲染

根据用户权限或设备类型选择性地渲染内容：

<!-- 根据用户权限选择性渲染 -->
{% if user_has_permission('view_detailed_stats') %}
  <div class="detailed-stats">
    {# 详细统计内容 #}
    {{ detailed_statistics_content }}
  </div>
{% else %}
  <div class="summary-stats">
    {# 摘要统计内容 #}
    {{ summary_statistics_content }}
  </div>
{% endif %}

<!-- 根据设备类型选择性渲染 -->
{% if is_mobile %}
  <div class="mobile-optimized">
    {# 移动优化内容 #}
    {{ mobile_content }}
  </div>
{% else %}
  <div class="desktop-optimized">
    {# 桌面优化内容 #}
    {{ desktop_content }}
  </div>
{% endif %}

7. 性能优化最佳实践总结

优化类别 最佳实践 性能提升
模板编译 启用缓存，预编译常用模板 50%-90%
上下文数据 预处理数据，减少模板内计算 30%-50%
模板结构 简化逻辑，合理使用宏和继承 20%-40%
循环优化 减少循环内计算，使用预处理列表 50%-80%
缓存策略 使用页面片段缓存，外部缓存系统 60%-95%
加载器优化 选择合适的加载器，预加载模板 20%-30%
重要提示：性能优化应该在实际测量的基础上进行，不同的应用场景可能需要不同的优化策略。始终先测量，再优化。
