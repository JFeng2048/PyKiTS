6. 环境配置特性

Jinja2环境配置提供了多种特性来控制模板的渲染行为。

空白控制

空白控制演示：

{% for i in range(3) %} 行 {{ i+1 }}: 前后空白已控制 {% endfor %}

自动转义

未转义HTML：

{{ unsafe_html }}

自动转义HTML：

{{ unsafe_html|safe }}

循环控制

使用扩展的循环控制：

    {% for skill in skills %} {% if skill == 'Java' %} {% continue %} {% elif loop.index > 3 %} {% break %} {% endif %}
  • {{ loop.index }}. {{ skill }}
    • {% endfor %}

Do语句

使用do语句操作变量：

{% set new_list = [] %} {% for skill in skills %} {% do new_list.append(skill.upper()) %} {% endfor %}

处理后的列表：{{ new_list }}

7. 上下文和环境配置最佳实践

上下文管理最佳实践

  • 组织数据：将相关数据组织成逻辑组（使用字典）
  • 命名约定：使用一致的命名约定，如蛇形命名法
  • 最小化全局变量：只包含真正需要全局访问的数据
  • 敏感数据处理：避免在模板中暴露敏感信息
  • 预处理数据：在传递给模板前尽可能处理数据

环境配置最佳实践

  • 开发vs生产：为不同环境使用不同的配置
  • 缓存策略：生产环境启用缓存，开发环境禁用缓存
  • 自动转义：始终启用自动转义以防止XSS攻击
  • 扩展使用：合理使用扩展增强模板功能
  • 自定义功能：将常用功能封装为过滤器、测试或全局函数
